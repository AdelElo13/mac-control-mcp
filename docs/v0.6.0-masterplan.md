## v0.6.0 scope (HARD cap — 14 items, per Codex split)

### A. Architectural (6)

**A1. Image-size discipline — redesigned per Codex critique**

v1 proposed `inline=false` default returning a local `/tmp` path. Codex
flagged this as (a) contract-break for existing clients and (b) broken
across remote transport. v2 design:

- `inline` defaults to **true** (no contract break)
- Opt-in `inline=false` returns `{content_ref: <url-or-path>, bytes, hash}`
- **Remote-transport safe**: when the server is reachable via HTTP
  (v0.7.0 transport work), `content_ref` is an HTTPS URL backed by a
  handler at `/artifacts/<hash>.png`. For local stdio, it is a path
  under `~/.mac-control-mcp/artifacts/` (user-readable).
- `max_bytes` default 4MB (return `isError=true` if caller disables
  downscale AND image exceeds), `max_dimension` default 4000px
- Downscale happens AFTER any OCR/grounding pass, so we never degrade
  what the agent sees — only what the agent ships
- Every response gets `_schema: "mcp-image-v1"` so future schema bumps
  don't silently break callers
- Artifact lifecycle: stored TTL 1 hour, GC'd by a background actor
  when it boots; explicit `delete_artifact(hash)` tool for early
  cleanup

**A2. OTel 1.40.0 MCP semantic conventions**

- New `ObservabilityController` actor
- Every `callTool()` invocation wrapped in a CLIENT span
- Standard attrs: `mcp.method.name`, `mcp.session.id`,
  `jsonrpc.request.id`, `gen_ai.tool.name`, `gen_ai.operation.name`,
  `error.type`
- Mac-specific attrs: `mac_control.app_bundle_id`,
  `mac_control.ax_role`, `mac_control.permission_tier`
- Exports to `OTEL_EXPORTER_OTLP_ENDPOINT` if set; otherwise
  `~/.mac-control-mcp/traces.jsonl` (one line per span)
- Zero-overhead when BOTH env unset AND local file disabled via
  `MAC_CONTROL_MCP_TRACE_DISABLED=1`

**A3. `.well-known/mcp.json` Server Card — substrate-only**

- Static JSON at `Resources/well-known-mcp.json`, read at startup,
  served via `resources/list` MCP method as `well-known` resource
- Fields: `name`, `version`, `description`, `capabilities` (tool
  count per phase), `contact`, `license: MIT`,
  `permission_model: "tiered + hierarchical scopes"`,
  `transport: "stdio"`,
  `directory_submission_ready: false` ← Codex v2 blocker fix: we
  can't claim submission-ready without OAuth+DPoP + public HTTPS
- Static Card file will be moved to HTTPS-served in v0.7.0 when
  remote transport + OAuth+DPoP land; that's when the submission
  claim flips to `true`

**A4. Structured outputs consistency audit**

- Every tool call MUST return `structuredContent` (not just text)
- Codified via a new test: `AllToolsReturnStructuredContent` that
  iterates the registry, calls each with minimal args, and asserts
  structured shape
- Existing tools that return only text get shim handlers that wrap

**A5. CoreWLAN-native `wifi_scan` — replaces dead `airport` utility**

- Kill shell-out to
  `/System/Library/PrivateFrameworks/Apple80211.framework/Resources/airport`
  (removed in Sonoma 14.4 per Gemini research)
- New implementation: `CWWiFiClient.shared().scanForNetworks(
  withName: nil)` from CoreWLAN framework
- Requires no new entitlement for basic scan
- Graceful degrade: if CoreWLAN returns empty on devices where scan
  is blocked (locked-down fleets), tool returns
  `{ok: false, hint: "wifi scan blocked — check device management"}`

**A6. Hierarchical permission scopes**

- `PermissionGrant` gets `allowSubDelegation: Bool` (default true for
  back-compat with v0.5.x grants)
- `enforceIfEnabled` carries a parent-grant context; when a tool's
  handler calls into a sub-tool, the grant propagates only if
  `allowSubDelegation == true`
- New tool arg on `request_access`: `allow_sub_delegation: bool`
- Use case: agent gets `click` scope on Safari; Safari-internal tool
  calls (browser_eval_js) inherit. But agent with `click` on Safari
  cannot `trash_file("~/safe.txt")` even if a prompt-injection tries.

### B. Vision + AX fallback (3) — **Swift-only, no Python sidecar**

**B1. `ground(target, strategy?)` — mixture-of-grounding router**

- Input: `target` = description or selector, `strategy` =
  `"ax"|"ocr"|"auto"` (default auto)
- Swift-only flow:
  1. AX attempt: `find_elements` by role/title → if 1 exact match,
     return AX coords
  2. OCR attempt: `ocr_screen` + fuzzy match target text → return
     bbox center
  3. Hybrid: when AX returns >1 match, use OCR to disambiguate
- Returns `{strategy_used, x, y, confidence: 0..1, candidates: [...]}`
- **Screen2AX vision fallback is NOT in v0.6.0** — that needs Swift
  port or Python sidecar, both carry risk. Deferred to v0.7.0 behind
  flag `MAC_CONTROL_MCP_ENABLE_SCREEN2AX=1`

**B2. `ax_tree_augmented(pid, max_depth?)` — redesigned per Codex v2**

Codex v2 flagged the v2 design (per-AX-element OCR) as a **performance
trap**: dense AX trees with 100+ empty-label elements would each
trigger an OCR pass = multi-second latency. v3 design:

- Walk native AX tree as today, record every element's `frame`
- Do ONE `ocr_screen` pass over the visible region
- Geometric join: for each OCR-detected text bbox, find the AX
  element whose `frame` contains it (innermost match wins); attach
  the text as `inferred_label` with `source: "ocr_geometric"`
- Empty-label AX elements that have no overlapping OCR text stay as-is
- Net: 1 OCR call instead of N

**Known-risk acknowledgement (Codex v3)**: innermost-match geometric
join can false-attribute when two AX frames tightly overlap
(z-stacked modals, popovers). v0.6.0 mitigation: every `inferred_label`
is flagged with `confidence: 0..1` where overlapping-frame cases get
confidence ≤0.5 — downstream tools (`ground`) weight these lower. True
per-pixel z-order detection requires CALayer inspection (deferred).

**Executable CI gate (Codex v3 blocker fix)**: new test
`AXAugmentBenchmarkTests.swift` — runs `ax_tree_augmented` against a
fixed-content ControlZoo test-window (20 labeled + 20 unlabeled AX
elements) on Apple Silicon. Assertions:

| Metric | Threshold |
|---|---|
| End-to-end latency | ≤ 500 ms (wall clock) |
| OCR-call count | == 1 (ensures single-pass, no regression to per-node) |
| `inferred_label` attachment rate | ≥ 80% of unlabeled-but-visible elements |
| Regression ceiling | ≤ 120% of first green baseline (prevents silent slowdowns) |

The fixed-content ControlZoo window becomes the reference fixture
checked into `Tests/MacControlMCPTests/Fixtures/controlzoo-ocr.html`.
CI fails if any assertion fails. No 500ms target, no merge.

Output shape: same as v3 (tree + `source` + `confidence` flags), so
downstream tools (`find_element`, `ground`) don't change.

**B3. `diff_ax_snapshot(pid, since_ms?)`**

- Two calls = a snapshot pair
- First call: returns `{snapshot_id, ts_ms, node_count}`
- Second call with same pid: returns `{added: [nodes], removed: [ids],
  changed: [{id, attr, old, new}]}`
- Built on CFHash-based AXKey dedup we already have
- Lets agents observe "what changed after I clicked" without
  re-capturing the screen

### F. Audit + memory + undo + PII redaction (5)

**F1. `audit_log_append(event, metadata?)` + `audit_log_read(since?, filter?)`**

- Append-only log at `~/.mac-control-mcp/audit.jsonl`
- Every destructive tool call auto-appends an entry before calling
  the side-effecting code (pre-condition logging) + an entry after
  (result logging). Agents can also manually append.
- Read supports `since_ts`, `filter_tool_name`, `limit`

**F2. `undo_last_action(steps?)` — snapshot-based, redesigned per Codex v2**

Codex v2 flagged the v2 design (blind backspace of `type_text`) as
**state-corrupting** — if the focus shifts or cursor moves between
`type_text` and `undo`, the backspaces delete the wrong chars. v3
design: snapshot-then-restore.

Before every destructive tool call, a **pre-image snapshot** is taken
and pushed to the LRU queue:

| Tool | Snapshot taken | Undo action |
|---|---|---|
| `type_text` | Target AX `AXValue` + `AXSelectedTextRange` | `set_element_attribute AXValue` back + restore selection |
| `set_volume` | `system_info.audio.volume` + `muted` flag | restore both |
| `set_dark_mode` | prior mode bool | flip back |
| `move_window` | `(x, y)` from AX | move back |
| `resize_window` | `(w, h)` | resize back |
| `set_element_attribute` | prior attr value | set back |
| `set_brightness` | prior level | set back |
| `trash_file` | target path (Trash path deterministic) | AppleScript `tell Finder to make new alias of previous trashed item` → put_back |
| `wifi_set` | prior power state | set back |
| `set_focus_mode` | prior focus mode name + state | set back |
| `click`, `press_key`, `mouse_event` | no pre-image possible (state machine effects) | not reversible — marked as such in queue |
| File dialog actions | user-state, not machine-state | not reversible |

Queue depth: 20. `undo_last_action(steps=1)` pops the top entry and
invokes the reverse; `steps=N` iterates. Returns `{undone:
[{tool, ok, method}], failed: [{tool, reason}]}`.

The snapshot capture uses `get_element_attributes` before the call,
so it's 1 extra AX read per destructive call (<5ms each) — acceptable
overhead for the safety property.

**Known-risk acknowledgement (Codex v3)**: undo is BEST-EFFORT, not
transactionally atomic. Between snapshot and undo, external processes
(OS events, user activity, other apps) can change the target state.
v0.6.0 undo guarantees: *if nothing external modified the target
between the tool call and the undo, the pre-image is restored
exactly*. It does NOT guarantee: rollback across crashes, rollback
across app quits, rollback that overrides user actions. Documented
explicitly in tool descriptions.

**Executable CI gate (Codex v3 blocker fix)**: new test file
`UndoFidelityTests.swift` — for each reversible row in the table
above, runs:

```
1. capture pre-image via get_element_attributes
2. invoke the destructive tool
3. assert state != pre-image  (tool actually changed something)
4. invoke undo_last_action(steps=1)
5. assert state == pre-image  (full fidelity)
```

Per-tool assertions:

| Tool | Fidelity assertion |
|---|---|
| `type_text` | AXValue string equality + SelectedTextRange equal |
| `set_volume` | volume int equal ±0, muted bool equal |
| `set_dark_mode` | dark-mode bool equal |
| `move_window` | origin x/y integer equal |
| `resize_window` | size w/h integer equal |
| `set_element_attribute` | attr value equal (exact type match) |
| `set_brightness` | level equal ±0.01 |
| `trash_file` | file exists at original path, not in Trash |
| `wifi_set` | power state enum equal |
| `set_focus_mode` | focus-mode name + state equal |

Non-reversible tools (`click`, `press_key`, `mouse_event`) tested to
assert that `undo_last_action` on them returns
`failed: [{tool, reason: "not_reversible"}]` (structured rejection,
not silent no-op).

CI fails if any assertion fails. Merge gated.

**F3. `agent_memory_store(key, value, tags?)` + `agent_memory_recall(query, limit?)`**

- A-Mem pattern: memory as tool calls (not black-box side effect)
- Storage at `~/.mac-control-mcp/memory.jsonl` with `ts, key, value,
  tags, session_id`
- Recall does substring + tag match (no embedding in v0.6.0, that's
  v0.7.0)

**F4. `redact_pii_text(text, categories?)`**

- Regex-based: emails, phones (E.164 + US), SSN, credit-card (Luhn),
  API-key patterns (AWS, Stripe, GitHub tokens, Anthropic sk-ant,
  OpenAI sk-)
- Returns `{redacted_text, redactions: [{category, count, spans}]}`

**F5. `redact_image_regions(path, regions)`**

- Blur/black-out specified bounding boxes in an image
- Inputs: image path + list of `[x, y, w, h]`
- Output: new image file with same extension, regions blurred
- Useful before `screenshot → share to Slack/GitHub` flows
